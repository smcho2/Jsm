Git & Github

Git은 소프트웨어를 개발하는 기업의 핵심 자산인 소스코드를 효과적으로 관리할 수 있게 해주는 무료, 공개소프트웨어입니다.
소스코드를 주고 받을 필요 없이, 같은 파일을 여러 명이 동시에 작업하는 병렬 개발이 가능합니다.
팀 프로젝트가 아닌, 개인 프로젝트일지라도 GIT을 통해 버전 관리를 하면 체계적인 개발이 가능해지고, 프로그램이나 패치를 배포하는 과정도 간단합니다.
github는 버전 관리와 협업을 위한 코드 웹 호스팅 플랫폼으로, 언제, 어디서나 협업 프로젝트를 쉽게 진행할 수 있도록 돕는 역할을 합니다.
Git과 Github의 차이점은 Git은 버전 관리 '프로그램'이고 Github는 버전 관리, 소스 코드 공유, 분산 버전 제어 등등이 가능한 원격 저장소입니다.

Jekyll

텍스트 변환 엔진으로, 마크업 언어로 글을 작성하면 미리 정의해 놓은 규칙에 따라서 정적인 웹사이트를 만들어준다. 이 과정에서 원하는 기능들을 구현할 수 있다.
정적 웹사이트를 만들기 때문에, PHP 등의 서버 소프트웨어가 필요 없어, 매우 빠르고 가볍다. Jekyll로부터의 사이트 이동이 이루어지는 이유는 동적 웹 페이지이기 때문이라고 합니다
동적 웹페이지는 상황에 따라 서버에 저장되어있는 HTML에 데이터 추가/가공을 해서 보여주는 방법입니다.
동적 웹페이지는 한 페이지에서 상황/시간/사용자요청에 따라 다른 모습을 보여줄 수 있다는 장점이 있지만 상대적으로 보안에 취약하고 모습이 계속 변하기 때문에
검색 엔진 최적화가 어렵다고 합니다.
Markdown Markdown은 일반 텍스트 기반의 경량 마크업 언어이다. 마크업 언어란, 태그 등을 이용하여 문서나 데이터의 구조 등을 명기하는 언어중 한 가지입니다.
텍스트만으로 서식이 있는 문서들을 작성할 때 자주 사용되며, 다른 마크업 언어들에 비해 문법이 쉽고 간단한 것이 특징입니다.
HTML 등의 서식 문서들로 쉽게 변환되기 때문에 README 파일이나 온라인 게시물 등에 사용됩니다.
Markdown의 장점으로는 먼저 문법이 간결하고 쉽습니다.
거의 대부분의 것에 사용할 수 있습니다. (웹사이트, 문서, 메모, README 기술파일 등)
마크다운을 지원하는 플랫폼이 다양합니다. (Github, Notion, Discord, Dropbox Paper 등)
대부분의 환경에서 작성 및 수정이 가능합니다. (일반 Notepad에서도 가능)
텍스트로 저장되어 용량을 많이 차지하지 않습니다.
다양한 형태로 변환이 가능하다.
